# Спецификация требований к мобильному приложению "LFK Voice Coach"

## 1. Введение

### 1.1 Назначение документа

Настоящий документ содержит полную спецификацию требований к мобильному Android-приложению для проведения лечебной физической культуры (ЛФК) при остеохондрозе позвоночника, болях в шее и пояснице. Документ предназначен для использования командой разработки в качестве основы для проектирования и реализации приложения.

### 1.2 Область применения

Приложение разрабатывается для платформы Android и представляет собой интерактивного голосового тренера для выполнения комплекса лечебных упражнений. Основная целевая аудитория — люди, страдающие от остеохондроза позвоночника и болей в шее, нуждающиеся в регулярном выполнении комплекса лечебных упражнений.

### 1.3 Определения и сокращения

| Термин | Определение |
|--------|-------------|
| Блок | Группа упражнений одной направленности (шея, плечи, растяжка, кор) |
| Упражнение | Отдельная единица лечебной гимнастики с заданными параметрами выполнения |
| Повторение | Однократное выполнение упражнения |
| Задержка | Время удержания положения (holdTime) |
| Расслабление | Время отдыха между подходами (relaxTime) |
| Sides | Чередование сторон (правая/левая рука, нога) |
| TTS | Text-to-Speech — синтез речи из текста |
| STT | Speech-to-Text — распознавание речи |

## 2. Общее описание

### 2.1 Цель продукта

Разработать мобильное Android-приложение, которое выступает в роли интерактивного голосового тренера для выполнения лечебного комплекса упражнений при остеохондрозе позвоночника, болях в шее и пояснице. Приложение обеспечивает:

- Голосовое объявление текущего упражнения и инструкций по его выполнению
- Синхронный отсчёт времени выполнения и количества повторений
- Управление последовательностью упражнений посредством голосовых команд пользователя
- Контроль перерывов между упражнениями и чередование сторон

### 2.2 Функциональные возможности

#### 2.2.1 Основные функции

1. **Голосовое сопровождение упражнений**
   - Объявление названия блока и упражнения перед его началом
   - Озвучивание полной инструкции (fullText)
   - Проговаривание команд на вдох/выдох, движение
   - Отсчёт времени в формате «раз, два, три...»
   - Объявление завершения упражнения и количества выполненных повторений

2. **Управление последовательностью упражнений**
   - Ожидание голосовой команды пользователя для перехода к следующему упражнению
   - Возможность повтора упражнения по команде
   - Остановка тренировки по команде

3. **Конфигурация комплекса упражнений**
   - Загрузка уроков из каталога (assets / internal storage / server)
   - Поддержка чередования сторон (sides)
   - Настраиваемое количество повторений

4. **Голосовое взаимодействие**
   - Распознавание команд: «следующее», «повторить», «стоп», «пауза»
   - Подтверждение принятых команд голосовым ответом

### 2.3 Архитектура приложения

Приложение реализуется на основе чистой архитектуры с разделением на следующие слои:

- **Presentation Layer** — пользовательский интерфейс и ViewModel
- **Domain Layer** — бизнес-логика (WorkoutService, WorkoutIterator)
- **Data Layer** — репозитории и модели данных

## 4. Функциональные требования

### 4.1 Требования к голосовому модулю (TTS)

| ID | Требование | Приоритет |
|----|------------|-----------|
| TTS-001 | Приложение должно использовать встроенный Android TTS engine для синтеза речи | Критический |
| TTS-002 | Приложение должно проверять доступность TTS и загружать языковой пакет при первом запуске | Критический |
| TTS-003 | Приложение должно поддерживать русский язык синтеза речи | Критический |
| TTS-004 | Приложение должно озвучивать текст с естественной скоростью (speechRate = 1.0) | Высокий |
| TTS-005 | Приложение должно обеспечивать очередность озвучивания во избежание наложения фраз | Высокий |

### 4.2 Требования к распознаванию речи (STT)

| ID | Требование | Приоритет |
|----|------------|-----------|
| STT-001 | Приложение должно использовать Android Speech Recognition API | Критический |
| STT-002 | Приложение должно распознавать команды: «следующее», «повторить», «стоп», «пауза» | Критический |
| STT-003 | Приложение должно ожидать команду пользователя в периоды перерыва между упражнениями | Критический |
| STT-004 | Приложение должно подтверждать распознанную команду голосовым ответом | Средний |

### 4.3 Требования к модулю упражнений

#### 4.3.1 Голосовые команды для упражнений

Приложение должно генерировать следующие голосовые команды в процессе выполнения упражнения:

| Команда | Условие активации | Пример озвучки |
|---------|-------------------|----------------|
| ANNOUNCE_EXERCISE | Перед началом упражнения | «Блок 1. Упражнения для укрепления мышц шеи. Упражнение 1: Первое упражнение...» |
| EXERCISE_ACTION | Перед выполнением | «Повторение 1. Правая рука» |
| COUNTDOWN | Во время задержки | «Раз... два... три... четыре... пять... шесть... семь» |
| RELAX | После задержки | «Расслабляем» или «Вдох. Расслабляем» |
| SIDE_SWITCH | При чередовании сторон | «Расслабляем. Повторение 2. Левая рука» |

#### 4.3.2 Примеры голосовых сценариев

**Сценарий 1: Упражнение с задерж��ой**
```
Модуль: "Блок 1. Упражнения для укрепления мышц шеи. Упражнение 1: Первое упражнение.
         Лежа на спине, положить кисти на лоб. Надавливать лбом на кисти в течение 5-7 с.,
         расслабиться в течение 7-8 с. Повторим 10-12 раз."

Подход 1:
Модуль: "Повторение 1."
Модуль: "Раз... два... три... четыре... пять... шесть... семь."
Модуль: "Расслабляем."

[Повторяется 10-12 раз]

Модуль: "Упражнение выполнено."
Модуль: "Скажите „следующее" для перехода к следующему упражнению."
[Ожидание голосовой команды пользователя]
```

**Сценарий 2: Упражнение с чередованием сторон**
```
Модуль: "Упражнение 2: Второе упражнение. Повторим 10-12 раз."

Подход 1 - Правая рука:
Модуль: "Повторение 1. Правая рука"
Модуль: "Раз... два... три... четыре... пять... шесть... семь."
Модуль: "Расслабляем."

Подход 1 - Левая рука:
Модуль: "Расслабляем. Повторение 2. Левая рука"
...
```

## 7. Постусловия и критерии приёмки

### 7.1 Критерии успешной реализации

1. Приложение корректно запускается на устройствах с Android 7.0 и выше
2. Голосовой модуль TTS воспроизводит все команды без задержек
3. Модуль распознавания речи корректно распознаёт команды «следующее», «повторить», «стоп»
4. Приложение корректно переходит между упражнениями по голосовой команде пользователя
5. Приложение корректно чередует стороны (sides)
6. Счёт времени работает с точностью до 1 секунды
7. Приложение сохраняет состояние при прерывании и восстанавливает его после возврата

### 7.2 Сценарии приёмочного тестирования

| № | Сценарий | Ожидаемый результат |
|---|----------|---------------------|
| 1 | Пользователь выбирает урок | Загружается JSON, отображается название |
| 2 | Пользователь нажимает "Старт" | Объявляется первое упражнение |
| 3 | Выполнение упражнения с задержкой 7 сек | Голосовой отсчёт «��аз... два... три... семь» |
| 4 | Чередование сторон | Объявление «Правая рука», затем «Левая рука» |
| 5 | После завершения упражнения пользователь говорит «следующее» | Переход к следующему упражнению |
| 6 | Во время паузы пользователь говорит «стоп» | Остановка тренировки |
